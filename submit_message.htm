When you submit a search to Find Articles, you're searching a gigantic index containing documents from over 136,000 different journals and 7,250 publishers. But the system powering Find Articles doesn't actually host the published version of anything in its index. Even the publishers don't necessarily host the published version. When you click a link to access the full text, you're actually requesting access to a version that's hosted on a completely external system most likely owned by one of a few different content aggregators (like EBSCO, or ProQuest, or Gale). In order to get you the correct article, the system has to request it in just the right way using just the right data. Since there are no guarantees about data format, quality, or consistency from publisher to publisher and aggregator to aggregator, any data in the mix that doesn't quite match up can make the lookup request fail and block you from accessing your article.

The only way to fix these problems is to report them to the system vendor so that they can work with publishers and aggregators to fix the affected metadata. If we don't know about problems, we can't report them. So thank you for telling us! The more reports you submit the better the system works for everyone.
